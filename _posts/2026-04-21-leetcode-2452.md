> 前缀和配合哈希表高效查找满足条件的子数组，是区间问题的利器。

在日常生活中，我们经常会遇到"输错一个字母"的情况。比如在搜索引擎中输入"progragm"，搜索引擎会自动纠正为"program"。这个过程涉及的就是编辑距离的概念。编辑距离越小，两个字符串就越"相似"。今天我们要讨论的题目，就是关于如何在最多两次编辑内让两个字符串变得相同的算法问题。

**原文链接**：[Words Within Two Edits of Dictionary](https://leetcode.com/problems/words-within-two-edits-of-dictionary/)

给你两个字符串数组 queries 和 dictionary，每个数组中的单词仅由小写英文字母组成，且长度相同。

一次编辑可以：将 queries 中的某个单词的任意一个字符改变为另一个不同的字符。

从 queries 中找到所有满足条件的字符串：经过最多两次编辑后，可以与 dictionary 中某个单词相同。

返回所有满足条件的 queries 中的单词，返回顺序与 queries 中的顺序一致。

### 示例

**示例 1**：

```
输入：queries = ["word","note","ants","wood"], dictionary = ["wood","joke","moat"]
输出：["word","note","wood"]
解释：将 "word" 的 'r' 改为 'o' 可变成 "wood"；将 "note" 的 'n' 改为 'j'，'t' 改为 'k' 可变成 "joke"。
     "ants" 需要超过 2 次编辑；"wood" 无需编辑即匹配。
```

**示例 2**：

```
输入：queries = ["yes"], dictionary = ["not"]
输出：[]
解释：对 "yes" 进行任意两次编辑都无法变成 "not"。
```

### 约束条件

```
1 <= queries.length, dictionary.length <= 100
n == queries[i].length == dictionary[j].length
1 <= n <= 100
```

解题思路：

1. 首先理解"编辑距离"的概念。在本题中，编辑操作特指将一个字符改变为另一个不同的字符。
2. 关键点：编辑次数限制最多允许两次编辑，需要比较两个字符串在不同位置上的差异字符个数。
3. 长度相等：题目保证 queries 和 dictionary 中的单词长度相同，简化了问题处理。
4. 返回值顺序：结果需要保持与 queries 相同的顺序，遍历时按顺序处理。
5. 解题方案：暴力遍历每个 query 和 dictionary 的组合，使用 isValid 函数检查两个单词的编辑距离是否不超过 2，如果编辑距离 ≤ 2，则将该 query 加入结果集。

代码：

```cpp
#include <gtest/gtest.h>
#include <climits>
#include <vector>
#include <string>

using namespace std;

class Solution {
public:
    bool isValid(string s, string p) {
        int diffs = 0;
        for (int i = 0; i < p.size(); ++i) {
            if (s[i] != p[i]) {
                diffs++;
            }

            if (diffs > 2) {
                break;
            }
        }

        return diffs <= 2;
    }

    vector<string> twoEditWords(vector<string> &queries, vector<string> &dictionary) {
        vector<string> ret;
        for (int i = 0; i < queries.size(); ++i) {
            for (int j = 0; j < dictionary.size(); ++j) {
                if (isValid(queries[i], dictionary[j])) {
                    ret.push_back(queries[i]);
                    break;
                }
            }
        }

        return ret;
    }
};

TEST(Daily, 2452) {
    Solution s;
    auto queries = vector<string>{"word", "note", "ants", "wood"};
    auto dictionary = vector<string>{"wood", "joke", "moat"};

    auto ret = s.twoEditWords(queries, dictionary);
    EXPECT_EQ(ret.size(), 3);
}
```

复杂度分析：

- **时间复杂度**：O(n × m × k)，其中 n 是 queries 的长度，m 是 dictionary 的长度，k 是单词的长度。
- **空间复杂度**：O(1)，只使用了常数级别的额外空间（不考虑输出空间）。

扩展思考：

如果将编辑操作扩展为包括插入、删除字符，那么问题就会变得更加复杂，需要使用经典的动态规划算法（如Levenshtein距离）来求解。在实际应用中，编辑距离广泛应用于：

- **拼写检查**：判断用户输入的单词是否可能是某个正确单词的变体
- **DNA序列比对**：比较两条DNA序列的相似程度
- **抄袭检测**：检测两篇文章的相似度

本题虽然看似简单，但其中蕴含的编辑距离概念在计算机科学中有广泛的应用。通过本题的学习，我们可以更好地理解字符串比较的原理，为处理更复杂的字符串问题打下基础。
