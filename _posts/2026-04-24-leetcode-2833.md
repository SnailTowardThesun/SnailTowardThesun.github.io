## 前言：赌徒的直觉

想象你在玩一个棋盘游戏。棋盘上只有一条线，起点是 0。每次掷骰子，你要么向左走一步（L），要么向右走一步（R）。但今天有个特别的骰子——它有时会出现一个 **通配符（_）**，你可以自己决定把它当 L 还是 R。

现在问题来了：**你最远能走到离起点多远的地方？**

直觉告诉我们：把所有通配符都投给占多数的那一边，这样就能一路冲刺到底。而令人惊讶的是，**这个直觉就是最优解**。

这道题（LeetCode 2833）表面上是一道贪心算法题，但实际上揭示了一个深刻的数学真理：**当所有选择都朝一个方向时，差异的绝对值就是最大距离**。

---

## 题目

**LeetCode 2833. Furthest Distance From Origin**

[原题链接](https://leetcode.com/problems/furthest-distance-from-origin/)

题目描述：

给你一个字符串 `moves`，其中只包含字符 `'L'`、`'R'` 和 `'_'`。

字符串中的下划线 `'_'` 可以表示为 `'L'` 或 `'R'`。

请你返回从起点 `(0, 0)` 出发，执行完所有移动后的**最远欧几里得距离**。

---

## 示例

**示例 1：**
```
输入：moves = "L_RL__R"
输出：3
```
解释：将 `'_'` 全部视为 `'R'`，路径变为 `"RRRLRR"`，最终位置为 x = 3。所以答案是 3。

**示例 2：**
```
输入：moves = "____"
输出：4
```
解释：所有字符都是通配符，全部视为 `'R'`，最终位置为 x = 4。

**示例 3：**
```
输入：moves = "LLRR"
输出：0
```
解释：L 和 R 数量相等，无论怎么分配通配符（本题无通配符），最终都回到原点。

## 解题思路

### 核心洞察：贪心即最优

这道题最精妙的地方在于——**不需要考虑任何动态规划**。贪心策略直接给出最优解。

**关键观察：**
1. 移动只发生在 x 轴上（L = x-1，R = x+1），y 坐标始终为 0
2. `'_'` 是万能的：它可以变成 L，也可以变成 R
3. 要让最终位置离原点最远，就要让正方向和负方向**差异最大化**

### 数学推导

设：
- `L_count` = L 的数量
- `R_count` = R 的数量
- `M_count` = `_` 的数量（通配符数量）

**贪心策略：**
- 如果 `L_count >= R_count`：把所有 `_` 变成 `L`
  - 最终位置 = `(L_count + M_count) - R_count` = `L_count + M_count - R_count`
  
- 如果 `R_count > L_count`：把所有 `_` 变成 `R`
  - 最终位置 = `R_count + M_count - L_count`

两种情况可以统一为：

```
最终位置 = |L_count - R_count| + M_count
```

**为什么？**

考虑差值 `D = L_count - R_count`：
- 如果 `D > 0`，L 占优。把 `_` 全加给 L，新差值 = `D + M_count`
- 如果 `D < 0`，R 占优。把 `_` 全加给 R，新差值 = `-D + M_count = |D| + M_count`
- 如果 `D = 0`，相等。把 `_` 随便加给任意一方，差值 = `0 + M_count = |D| + M_count`

所以答案永远是：`|L - R| + M`

### 代码实现

```cpp
#include <string>
#include <algorithm>
#include <cmath>

using namespace std;

class Solution {
public:
    int furthestDistanceFromOrigin(string moves) {
        int diff = 0, wild = 0;  // 差值 和 通配符数
        for (char c : moves) {
            if (c == 'L') diff--;
            else if (c == 'R') diff++;
            else wild++;
        }
        return abs(diff) + wild;
    }
};
```

- **空间复杂度**：O(1)，只用了常数个变量

## 延伸思考

### 从博弈论看贪心

这道题其实可以看作一个**零和博弈**：L 和 R 是两个对手，`_` 是裁判手中的牌，可以发给任何一方。

在这个博弈中，最优策略是什么？

答案是：**裁判把所有牌给当前领先的一方**——这就是我们的贪心策略。为什么？因为这样做会**放大现有差距**，让结果离原点更远。

### 实际应用场景

这类"通配符最优分配"问题在现实中无处不在：
- **网络路由**：在多条路径中选择最优组合
- **负载均衡**：将请求分配到压力最小的服务器
- **金融对冲**：将风险敞口重新分配到最有利的方向

## 总结

LeetCode 2833 是一题**简单却不平凡**的题目。它告诉我们：

> 有时，最优解不是最复杂的，而是最**诚实**的。

当我们面对不确定性（通配符）时，最诚实的策略就是**顺大势而行**——把资源全部投入到已经占优的方向。这种"滚雪球"式的贪心策略，在数学上是严格正确的，在哲学上也是值得品味的。
