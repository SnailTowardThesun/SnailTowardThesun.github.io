## 前言：当数字开始社交

想象一下，数组中的每个数字都是一位居民，住在一排连续的门牌号码上（索引即门牌）。每个人想要知道：**我和所有跟我同名的人之间的距离总和是多少？**

在现实生活中，如果你想知道自己和所有叫"张伟"的人之间的距离，你可能需要打开地图 App，一个一个量。但在数组的世界里，我们可以用一种优雅的方式——**前缀和**——一次性算出所有人的答案。

这道题的核心挑战在于：对于每个元素，需要计算它到所有与它值相同的元素的下标差的绝对值之和。暴力解法的时间复杂度是 O(n²)，而使用**前缀和优化**，我们可以将复杂度降至 **O(n)**。

---

## 题目

**LeetCode 2615. Sum of Distances**

https://leetcode.com/problems/sum-of-distances/

题目描述：

给定一个下标从 0 开始的整数数组 `nums`。对于每个下标 `i`，计算 `nums[i]` 的所有下标 `j`（其中 `j != i` 且 `nums[j] == nums[i]`）的绝对差之和，即 `|i - j|` 的总和。返回结果数组 `answer`。

---

## 示例

**示例 1：**
```
输入：nums = [1, 3, 1, 1, 2]
输出：[5, 0, 3, 4, 0]
```

解释：
- `i=0`：值为 1，相同值在索引 2 和 3 处。`|0-2| + |0-3| = 2 + 3 = 5`
- `i=1`：值为 3，无其他相同值。`0`
- `i=2`：值为 1，相同值在索引 0 和 3 处。`|2-0| + |2-3| = 2 + 1 = 3`
- `i=3`：值为 1，相同值在索引 0 和 2 处。`|3-0| + |3-2| = 3 + 1 = 4`
- `i=4`：值为 2，无其他相同值。`0`

**示例 2：**
```
输入：nums = [2, 2, 2, 2]
输出：[6, 4, 4, 6]
```

## 解题思路

### 关键观察

1. **分组**：只有值相同的元素之间才需要计算距离。我们可以用哈希表将相同值的索引分组。
2. **前缀和的力量**：对于一组已排序的索引 `[p0, p1, p2, ..., pm-1]`，计算每个索引到其他所有索引的距离和，看似需要 O(m²)，但利用**前缀和**可以降至 O(m)。

### 核心推导

假设某个值的所有下标已排序为 `[p0, p1, ..., pm-1]`。对于位置 `i` 处的下标 `pi`：

- **左侧距离和** = 所有左侧下标到 pi 的距离 = `(p1 - p0) + (p2 - p0) + ... + (pi - p0)` = `i × pi - (p0 + p1 + ... + p(i-1))`
- **右侧距离和** = 所有右侧下标到 pi 的距离 = `(pi+1 - pi) + (pi+2 - pi) + ... + (pm-1 - pi)` = `(p(i+1) + ... + pm-1) - (m - 1 - i) × pi`

令：
- `prefix_sum = p0 + p1 + ... + p(i-1)`（左侧下标之和）
- `suffix_sum = (p0 + p1 + ... + pm-1) - prefix_sum - pi`（右侧下标之和）
- `total_sum = p0 + p1 + ... + pm-1`（全部下标之和）

则：
- **左侧距离和** = `i × pi - prefix_sum`
- **右侧距离和** = `suffix_sum - (m - 1 - i) × pi`

### 逐步推导示例

以 `nums = [1, 3, 1, 1, 2]` 为例，值为 1 的下标列表为 `[0, 2, 3]`，`m = 3`：

| i | pi | prefix_sum | 左侧 = i×pi - prefix | suffix_sum | 右侧 = suffix - (m-1-i)×pi | 总距离 |
|---|----|------------|---------------------|------------|--------------------------|--------|
| 0 | 0  | 0（初始值）| 0×0 - 0 = **0**     | 5          | 5 - 2×0 = **5**          | 0+5=**5** |
| 1 | 2  | 0          | 1×2 - 0 = **2**     | 3          | 3 - 1×2 = **1**          | 2+1=**3** |
| 2 | 3  | 2          | 2×3 - 2 = **4**     | 0          | 0 - 0×3 = **0**          | 4+0=**4** |

*注意：i=0 时 prefix_sum 初始为 0，左侧无元素，故左侧贡献为 0。*

完美匹配！

### 算法步骤

1. 遍历数组，用哈希表将相同值的索引分组
2. 对于每个值对应的下标列表：
   - 若只有一个元素，跳过（距离为 0）
   - 计算所有下标之和 `total_sum`
   - 遍历下标列表，维护 `prefix_sum`，按公式计算每个位置的左右距离
3. 返回结果数组

## C++ 代码实现

```cpp
#include <unordered_map>
#include <vector>

using namespace std;

class Solution {
public:
    vector<long long> distance(vector<int>& nums) {
        // 步骤1：用哈希表分组相同值的索引
        unordered_map<int, vector<long long>> positions;
        for (long long i = 0; i < nums.size(); ++i) {
            positions[nums[i]].push_back(i);
        }

        vector<long long> answer(nums.size(), 0);

        // 步骤2：对每组下标计算距离和
        for (auto& [value, pos] : positions) {
            if (pos.size() <= 1) continue;

            // 计算所有下标之和
            long long total_sum = 0;
            for (auto& idx : pos) {
                total_sum += idx;
            }

            // 遍历每个下标，利用前缀和计算左右距离
            long long prefix_sum = 0;
            int m = pos.size();
            for (int i = 0; i < m; ++i) {
                long long cidx = pos[i];

                // 左侧距离和 = 左侧元素个数 × 当前位置 - 左侧元素下标和
                long long left = i * cidx - prefix_sum;

                // 右侧元素下标和
                long long suffix_sum = total_sum - prefix_sum - cidx;
                // 右侧距离和 = 右侧元素下标和 - 右侧元素个数 × 当前位置
                long long right = suffix_sum - (m - 1 - i) * cidx;

                answer[cidx] = left + right;

                prefix_sum += cidx;
            }
        }

        return answer;
    }
};
```

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 是数组长度。每个元素在哈希表构建和前缀和计算中各被访问一次。
- **空间复杂度**：O(n)，用于存储哈希表。`answer` 数组作为题目要求的返回值，不计入额外空间复杂度。

## 延伸思考

### 数学之美

当一群数字排成一列，寻找彼此之间的距离——这看似是数组运算，实则是**有序性**的优雅应用。当索引列表自然有序时，前缀和技巧可以在 O(1) 时间内计算出每个点的加权距离和，无需暴力遍历。

### 实际应用

- **日志分析**：在同一类事件流中快速计算事件的时间间隔之和
- **数据库查询优化**：计算相同条件的查询在日志中的分布密度
- **推荐系统**：分析用户在相同行为模式下的时间间隔，用于特征工程

## 总结

前缀和优化是这道题的灵魂所在。它将看似需要 O(m²) 的距离计算压缩到了 O(m)，是处理同类元素距离问题的经典技巧。掌握这个思路，面对类似问题时，就能从暴力解法跃升至优雅的最优解。
