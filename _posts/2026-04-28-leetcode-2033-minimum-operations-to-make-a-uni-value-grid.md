## 题目

LeetCode 2033. Minimum Operations to Make a Uni-Value Grid (获取单值网格的最小操作数)

Difficulty: **Medium**

给你一个二维整数网格 `grid`，`grid[i][j]` 是第 `i` 行第 `j` 列的单元格价值。

你可以对 `grid` 中的任意元素执行以下操作任意次（也可以零次）：
- 如果 `grid[i][j] >= x`，则将 `grid[i][j]` 减少 `x`
- 如果 `grid[i][j] <= x`，则将 `grid[i][j]` 增加 `x`

如果可以将网格 `grid` 转换为单值网格（所有元素都相同的网格），返回最少操作次数。否则，返回 `-1`。

### 示例

```
示例 1：
输入：grid = [[2,4],[6,8]], x = 2
输出：4
解释：
可以执行以下操作使所有元素变为 4：
- grid[0][0] 从 2 增加 2 次 2 -> 4（2次操作）
- grid[0][1] 从 4 减少 0 次 2 -> 4（0次操作）
- grid[1][0] 从 6 减少 1 次 2 -> 4（1次操作）
- grid[1][1] 从 8 减少 2 次 2 -> 4（2次操作）
总计 4 次操作
```

```
示例 2：
输入：grid = [[1,2],[3,4]], x = 2
输出：-1
解释：1%2=1, 2%2=0, 3%2=1, 4%2=0，无法统一
```

## 解题思路

这是一道典型的**中位数贪心**问题。核心思路是将所有元素调整到中位数时，操作次数最少。

### 核心思路

1. **可行性判断**：所有元素对 `x` 取余必须相同，否则无法统一，返回 `-1`
2. **展平排序**：将二维网格展平为一维数组并排序
3. **选择中位数**：中位数是最优目标值，能使总操作次数最小
4. **计算总操作次数**：所有元素到中位数的距离之和除以 `x`

### 为什么选择中位数

**中位数最小化绝对差之和**是一个经典的数学结论：

> 在数轴上，所有点到中位数的绝对距离之和最小。

**直观理解**：
- 如果选择比中位数小的值作为目标，右边的点需要移动更远
- 如果选择比中位数大的值作为目标，左边的点需要移动更远
- 中位数恰好平衡了左右两边的移动距离

### 可行性判断的原理

每个元素可以通过加减 `x` 来调整，这意味着所有元素必须属于同一个**模 x 等价类**。即：

```
grid[i][j] = k * x + r
```

其中 `r` 是余数（`0 <= r < x`）。要使所有元素相等，它们的余数 `r` 必须相同。

## 代码实现

{% raw %}
```cpp
class Solution {
public:
    int minOperations(vector<vector<int>>& grid, int x) {
        int m = grid.size();
        int n = grid[0].size();

        int t = grid[0][0] % x;

        vector<int> container(m*n, 0);
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] % x != t) {
                    return -1;
                }
                container[i*n+j] = grid[i][j];
            }
        }

        sort(container.begin(), container.end());
        int median = container[m * n / 2];

        int ret = 0;
        for (auto i : container) {
            ret += abs(i - median);
        }

        return ret / x;
    }
};
```
{% endraw %}

### 代码解析

1. **可行性检查**（第 10-18 行）：遍历网格，检查每个元素的余数是否与第一个元素相同
2. **展平存储**（第 12、17 行）：将二维网格元素按行优先顺序存入一维数组
3. **排序找中位数**（第 20-21 行）：排序后取中间位置的元素作为中位数
4. **计算操作次数**（第 23-26 行）：累加所有元素到中位数的距离，最后除以 `x`

### 复杂度分析

- **时间复杂度**：O(m*n*log(m*n))，主要耗时在排序操作
- **空间复杂度**：O(m*n)，用于存储展平后的数组

## 总结

这道题的核心在于：
1. **模运算判断可行性**：所有元素必须同余才能统一
2. **中位数贪心策略**：选择中位数作为目标值最小化总操作次数
3. **操作次数计算**：距离之和除以 `x` 得到最终结果

通过排序和中位数选择，可以在 O(m*n*log(m*n)) 的时间复杂度内解决问题。这道题很好地体现了数学思维在算法中的应用。
