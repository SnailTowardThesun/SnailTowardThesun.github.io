判断通过首次操作是否能使数组递增：数学分析，贪心策略。

题目：

给你两个字符串 s1 和 s2，长度都是 4，只包含小写英文字母。
你可以对任意一个字符串执行以下操作任意次：
- 选择两个下标 i 和 j，满足 i < j 且 j - i 是偶数，然后交换这两个下标的字符
如果可以让 s1 和 s2 相等，返回 true，否则返回 false。

示例：

示例 1：
输入: s1 = "abcd", s2 = "cdab"
输出: true
解释: 交换 s1 的 0 和 2 位置字符得到 "cbad"，再交换 1 和 3 位置字符得到 "cdab"
示例 2：
输入: s1 = "abcd", s2 = "abcd"
输出: true
解释: 无需任何操作，两个字符串已经相等

题目翻译：

给你两个字符串 s1 和 s2，长度都是 4，只包含小写英文字母。
你可以对任意一个字符串执行以下操作任意次：
- 选择两个下标 i 和 j，满足 i < j 且 j - i 是偶数，然后交换这两个下标的字符
如果可以让 s1 和 s2 相等，返回 true，否则返回 false。

解题思路：

1. 关键观察：
- 操作要求 j - i 是偶数，这意味着 i 和 j 必须具有相同的奇偶性
- 即：偶数位置(0,2)的字符只能在偶数位置间交换
- 奇数位置(1,3)的字符只能在奇数位置间交换
2. 算法步骤：
- 分别提取 s1 和 s2 的偶数位置字符(0,2)和奇数位置字符(1,3)
- 对两组字符分别排序后比较
- 如果两组都相等，则可以通过操作使 s1 和 s2 相等
3. 简化实现：
- 由于字符串长度固定为4，可以直接尝试所有可能的交换
- 偶数位置(0,2)可以交换，奇数位置(1,3)可以交换
- 检查所有组合是否能得到 s2
4. 复杂度分析：
- 时间复杂度: O(1)，固定长度的字符串，操作次数有限
- 空间复杂度: O(1)，只使用常数额外空间

代码：

```cpp
Solution s;
    string s1 = "abcd";
    string s2 = "cdab";
    auto ret = s.canBeEqual(s1, s2);
    EXPECT_EQ(true, ret);
};
```
