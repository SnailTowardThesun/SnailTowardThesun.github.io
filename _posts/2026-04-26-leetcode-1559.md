## 前言：迷宫中的自我循环

想象你走进一座由相同字符构成的迷宫。每一步你都能向左、右、上、下四个方向移动，但只能走在与自己脚下字符相同的地砖上。

现在的问题是：**你能否走出一个环？** 换句话说，是否存在一条路径，让你从某个格子出发，沿着相同字符的路径走一圈，最终回到起点？

这听起来像是一个经典的迷宫问题，但实际上它揭示了图论中一个核心概念——**环检测**。

LeetCode 1559 题（Detect Cycles in 2D Grid）正是要求我们判断一个二维字符网格中是否包含由相同字符组成的环。

---

## 题目

**LeetCode 1559. Detect Cycles in 2D Grid**

[原题链接](https://leetcode.com/problems/detect-cycles-in-2d-grid/)

题目描述：

给你一个 `m x n` 的二维字符网格 `grid`，请你判断其中是否包含由相同字符值组成的**环**。

**环的定义：**
1. 环由相同字符值组成，路径长度**至少为 4**。
2. 沿着网格中的四个方向（左、右、上、下）移动形成。
3. 移动过程中**不能往回走**（即不能沿原边返回）。
4. 同一个格子可以在环中被访问多次，但**边不能重复**。

---

## 示例

**示例 1：**
```
输入：grid = [
  ['a','a','a','a'],
  ['a','b','b','a'],
  ['a','b','b','a'],
  ['a','a','a','a']
]
输出：true
```
解释：沿着边缘的 `'a'` 形成了一个完整的环。路径长度为 12（>= 4），满足条件。

**示例 2：**
```
输入：grid = [
  ['a','b','b'],
  ['b','z','z'],
  ['z','z','f']
]
输出：false
```
解释：没有任何相同字符构成的环。注意虽然 `'b'` 和 `'z'` 各自有连通区域，但没有形成环。

## 解题思路

### 从问题本质看：图的环检测

这道题可以看作一个**隐式图**。每个格子是一个节点，相邻且字符相同的格子之间存在边。问题转化为：**这张图中是否存在环？**

### 核心策略：DFS + 路径记忆

我们使用深度优先搜索（DFS）来遍历每个连通分量，并在遍历过程中检测环。

**关键洞察：**
1. **visited 数组**：记录每个格子是否被访问过，避免重复遍历。
2. **父节点追踪**：DFS 时需要知道从哪个格子来的，避免沿着原路返回。
3. **环的检测**：如果当前格子遇到一个**已访问过且不是父节点**的邻居，说明找到了环。

### 算法步骤

1. 初始化 `visited` 数组，全部标记为未访问。
2. 遍历网格中的每个格子：
   - 如果该格子未访问，从它开始 DFS。
   - DFS 过程中，沿四个方向查找相同字符的相邻格子。
   - 如果遇到已访问且不是父节点的邻居 → **找到环**。
   - 如果 DFS 结束没有发现环，继续遍历下一个未访问格子。
3. 如果所有连通分量都没有环，返回 `false`。

### 一个易错点

DFS 的起始调用需要特殊处理父节点参数。通常父节点设为 `(-1, -1)`，因为合法的坐标不会是这个值，这样可以正确区分"初始节点"和"已发现的环"。

## 代码实现

```cpp
#include <vector>

using namespace std;

class Solution {
    int m_, n_;
    vector<vector<bool>> vis_;
    // 四个方向：左、右、上、下
    const int DIRS[4][2] = {{0, -1}, {0, 1}, {-1, 0}, {1, 0}};

    /**
     * @brief DFS 检测是否存在环
     * @param x  当前行坐标
     * @param y  当前列坐标
     * @param px 父节点行坐标
     * @param py 父节点列坐标
     * @param c  当前字符值
     * @param grid 输入网格
     * @return 找到环返回 true
     */
    bool dfs(int x, int y, int px, int py, char c, const vector<vector<char>>& grid) {
        vis_[x][y] = true;
        for (auto& d : DIRS) {
            int i = x + d[0], j = y + d[1];
            // 边界检查和字符匹配
            if (i >= 0 && i < m_ && j >= 0 && j < n_ && grid[i][j] == c) {
                if (vis_[i][j]) {
                    return true;  // 找到环！
                }
                // 不是父节点才递归
                if ((i != px || j != py) && dfs(i, j, x, y, c, grid)) {
                    return true;
                }
            }
        }
        return false;
    }

public:
    bool containsCycle(vector<vector<char>>& grid) {
        m_ = grid.size(), n_ = grid[0].size();
        vis_.assign(m_, vector<bool>(n_, false));

        for (int i = 0; i < m_; i++) {
            for (int j = 0; j < n_; j++) {
                if (!vis_[i][j]) {
                    if (dfs(i, j, -1, -1, grid[i][j], grid)) {
                        return true;
                    }
                }
            }
        }
        return false;
    }
};
```

## 延伸思考

### 为什么 DFS 能检测环？

在图的遍历中，**环的存在等价于存在一条"前向边"或"交叉边"**。DFS 的特性是它会沿着一条路径深入探索，当遇到一个已经访问过的节点时，如果这个节点不是当前路径上的祖先（在 DFS 树中），那就意味着存在环。

在我们的实现中，`vis_` 数组充当了"已访问"标记。当 DFS 遇到一个已访问的邻居，且它不是父节点时，就说明这条路径和之前探索的路径形成了一个闭环。

### 环长度 >= 4 的约束

题目要求环的长度至少为 4。在网格图中，这意味着**最小的环是一个 2x2 的正方形**。因为 DFS 检测到的任何环至少需要 4 条边才能闭合（每个方向至少走一步），所以只要检测到环，就自动满足长度 >= 4 的条件，无需额外检查。

### BFS 可行吗？

理论上 BFS 也可以检测环，但实现起来稍复杂。BFS 需要记录每个节点的"层"和"父节点"，在遇到已访问节点时需要更复杂的逻辑来判断是否构成环。DFS 的递归天然携带了路径信息，更适合这个场景。

## 总结

LeetCode 1559 是一道**中等难度的图论入门题**。它教会我们：

> 环检测的本质不是找"圈"，而是发现"两条不同的路径通向同一个地方"。

在 DFS 的框架下，`visited` 数组是我们的记忆，`父节点`是我们来时的路。当记忆中发现了一个既不是起点、也不是来路的节点时——环，就在那里。

这道题的优雅之处在于：**不需要 Tarjan 算法，不需要并查集**，简单的 DFS 遍历加上路径追踪就足够了。有时，最朴素的方法恰恰是最有效的。
